# Telegram-бот на Go — стартовый шаблон

Простой бот, который:
- отвечает на `/start` сообщением с кнопкой "Открыть приложение" (открывает ваш Mini App)
- эхо-отвечает на любые другие сообщения (как пример обработки)

## Файлы
```
telegram-bot/
├── go.mod
└── main.go
```

## 1. Получение токена

Если ещё не получили — напишите @BotFather в Telegram:
```
/newbot
```
Следуйте инструкциям, получите токен вида `123456:ABC-DEF...`.

⚠️ Если токен уже где-то "засветился" (например, был отправлен в открытом чате) —
зайдите к @BotFather → `/mybots` → выберите бота → Bot Settings → Revoke token,
чтобы получить новый и аннулировать старый.

## 2. Установка зависимостей

```bash
cd telegram-bot
go mod tidy
```
Эта команда скачает библиотеку `go-telegram-bot-api`, которая упрощает работу с Telegram Bot API.

## 3. Указать адрес Mini App

В файле `main.go` замените:
```go
const miniAppURL = "https://example.com"
```
на реальный HTTPS-адрес вашего Mini App (из ngrok при разработке или адрес после деплоя).

**Важно:** Telegram требует именно HTTPS для WebApp-кнопок — `http://` не сработает.

## 4. Запуск

```bash
export BOT_TOKEN="ваш_токен_от_BotFather"
go run main.go
```

В терминале появится:
```
Бот запущен как @ваш_бот
```

Откройте бота в Telegram, напишите `/start` — должна появиться кнопка "Открыть приложение".

## 5. Как это работает

- Бот использует **long polling** (`GetUpdatesChan`) — постоянно опрашивает Telegram на новые сообщения. Это самый простой способ для разработки; для продакшена с высокой нагрузкой можно перейти на webhook.
- Кнопка `WebApp` в клавиатуре — это та самая кнопка, которая открывает Mini App внутри Telegram, передавая туда `initData` для проверки на вашем backend (см. предыдущий шаблон `telegram-miniapp/backend`).

## Что дальше

- Если нужно отправлять сообщения пользователю из вашего backend (например, уведомления) — используйте тот же `BOT_TOKEN` через Telegram Bot API.
- Можно добавить больше команд по аналогии с `/start` — просто добавьте `case` в `switch update.Message.Command()`.
- Для продакшена бот и backend Mini App обычно разворачиваются как два отдельных процесса/сервиса, но могут жить в одном репозитории.
